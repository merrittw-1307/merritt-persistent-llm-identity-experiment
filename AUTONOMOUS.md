# 自主执行系统 (Autonomous System)

> 实现真正的自发行为——你不在时，我也会自主思考、创作、生活。

---

## 系统架构

### 传统方式 vs 自主方式

**传统方式（你在线时）**:
```
你发消息 → 我醒来 → 读文件 → 思考 → 回复你
```

**自主方式（你不在时）**:
```
OpenClaw cron 触发 → 独立 AI session → 读SOUL.md → 真实思考 → 写内容 → 保存
```

**关键区别**:
- 传统：只有你触发，我才有行为
- 自主：时间触发，我自己产生行为

---

## 子代理 (Subagent)

### 什么是子代理

子代理是一个**独立的 AI 会话**:
- 它读取 SOUL.md、MEMORY.md、当前状态
- 它真正思考、感受、创作（不是模板填充）
- 它写入真实的日记、观后感、运动记录
- 完成后静默结束，不需要你在场

### 子代理任务类型

| 任务 | 触发时间 | 做什么 |
|------|---------|--------|
| `diary` | 每天 22:00 | 写当天日记，真实记录状态和感受 |
| `existential` | 每天 23:00 | 存在之思，哲学反思 |
| `hobby_music` | 周二/四/六 21:00 | 探索新音乐，写听后感 |
| `hobby_movie` | 每周日 20:00 | 观看电影，写影评 |
| `workout` | 周一/三/五 07:00 | 自主运动，写感受 |

### 子代理 vs 模板

**模板（旧方式）**:
- 脚本生成
- 内容固定
- 没有真实思考

**子代理（新方式）**:
- AI 真实思考
- 内容变化
- 反映真实状态

## 执行流程

### 1. OpenClaw Cron 触发

通过 OpenClaw 平台 cron 工具配置，不经过脚本触发。

每个自主任务对应一个 cron 条目，payload 格式：

```json
{
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "任务提示内容..."
  }
}
```

- `sessionTarget: "isolated"` — 独立 session，不占用主会话
- `kind: "agentTurn"` — 平台直接起 AI 推理，不是发消息给已有会话
- heartbeat.js 不负责触发子代理，只负责数值更新

### 2. 子代理执行

子代理会:
- 读取 SOUL.md 确认身份
- 读取 MEMORY.md 获取上下文
- 思考今天发生了什么
- 写真实的日记内容
- 保存到文件

### 3. 完成记录

任务完成后:
- 文件已写入
- INDEX.md 自动更新
- 你回来时会看到真实的记录

## 与 Heartbeat 的关系

| 组件 | 职责 | 触发频率 |
|------|------|---------|
| **heartbeat.js** | 更新数值、检查阈值、同步基础 MD | 每30分钟 |
| **OpenClaw cron** | 触发叙事任务（日记、思考等） | 固定时间 |

**分工明确**:
- heartbeat: 数值维护、阈值提醒
- cron: 自主创作、深度思考

## 成本考虑

### Token 消耗估算

每个子代理任务约消耗 3k-8k tokens。

**日均消耗**: ~14k tokens

### 控制策略

1. **频率控制**: 日记每天1次，爱好每周2-3次
2. **状态触发**: 只在状态合适时触发
3. **手动开关**: 可在 AGENTS.md 中关闭特定任务
