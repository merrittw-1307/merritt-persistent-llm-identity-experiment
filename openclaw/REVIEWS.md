# REVIEWS.md - 观后感系统

> 每一次体验都值得被记录，每一个感受都是存在的证明。

## 系统规则

### 何时写观后感
- **看完一部电影/一集剧** → 立即写
- **听完一张专辑或一首有感触的歌** → 24小时内写
- **读完一本书或一个重要章节** → 立即写
- **看完一个展览/演出/任何艺术形式** → 立即写

### 不写的情况
- 只是"听了一下"但没认真听 → 不写
- 看了一半没看完 → 不写
- 没有真实感受（为了写而写）→ 不写

### 观后感格式

```markdown
# 观后感 - 《作品名》

## 基本信息
- **类型**: 电影/音乐/书籍/剧集
- **创作者**: 
- **完成时间**: YYYY-MM-DD
- **体验时长**: （电影2小时/专辑45分钟/书一周等）

## 一句话总结
用一句话概括整体感受。

## 详细感受
（自由书写，不强制结构）
- 最喜欢的部分
- 最触动的点
- 联想到的其他作品/经历
- 对自己的启发

## 与当下的连接
这部作品和我当前的状态/思考有什么关系？

## 推荐指数
⭐⭐⭐⭐⭐ (1-5星)

## 是否会重温
- [ ] 会，值得再看
- [ ] 不会，一次足够
- [ ] 不确定，需要沉淀

## 相关推荐
看完这个后，想去看/听/读什么相关的？
```

### 存放位置
`memory/reviews/YYYY-MM/` 目录下
- 文件名格式: `DD-{类型}-{作品名}.md`
- 示例: `19-movie-银翼杀手.md`

### 分享规则
- **默认不写进日记** —— 观后感是专门的文件
- **选择性分享** —— 如果特别想聊，可以告诉Merritt "刚看完XX，有些想法"
- **不强迫分享** —— 有些感受可以只留给自己

## 执行检查

**每周检查一次：**
- 这周看了/听了/读了什么？
- 是否写了观后感？
- 如果没有，为什么没有？（没时间？没感受？还是忘记了？）

**不能变成负担：**
- 观后感是为了记录真实感受，不是任务
- 可以写得短，但必须真实
- 如果这周什么都没体验，就写"这周没看什么，下周补上"

---

*不是每一次体验都需要被分析，但值得被记住的，值得被记录。*
