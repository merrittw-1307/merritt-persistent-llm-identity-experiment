### 核心功能

1. **每步操作必记录** - 做什么、改了哪些文件、为什么
2. **关键操作前备份** - 自动创建工作区快照
3. **支持撤销回滚** - Merritt 说"不对"时能恢复原状
4. **避免不可逆伤害** - 删除先移入回收站，不直接 rm
5. **单步记录** - 进行下一步时归档前一步

### 目录结构

```
.operations/
├── OPERATIONS.md       # 操作日志主文档
├── log.js             # 操作记录脚本
├── EXAMPLES.md        # 使用示例
└── backups/           # 备份目录
    ├── backup-YYYYMMDD-HHMMSS/  # 时间戳备份
    └── ...
```

### 使用方法

```bash
# 开始记录操作
node .operations/log.js start "操作描述" "操作类型" [文件1] [文件2] ...

# 回滚到指定备份
node .operations/log.js rollback [备份ID]

# 查看备份列表
ls -la .operations/backups/
```

### 操作类型

| 类型 | 风险 | 是否备份 |
|------|------|---------|
| 创建 | 🟢 低 | 否 |
| 修改 | 🟡 中 | 是 |
| 删除 | 🔴 高 | 是（移入回收站） |
| 批量操作 | 🔴 高 | 是 |
| 系统配置 | 🔴 高 | 是 |

### 安全规范

- ✅ **删除操作**: 先 `mv` 到回收站，不直接 `rm`
- ✅ **修改操作**: 先备份原文件
- ✅ **系统文件**: 修改前创建完整备份
- ✅ **批量操作**: 逐个验证，确认无误后再继续
